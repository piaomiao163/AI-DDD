# 招投标管理系统 — 完整产品需求文档

> 版本：v1.0 | 日期：2026-06-27 | 作者：产品设计团队

---

## 一、产品定位

**产品名称：** BidFlow — 智能招投标管理平台

**核心价值主张：**
> 让招投标过程透明、合规、高效。从项目立项到合同签署，全流程线上化管控，AI 辅助评标降低人为干预风险。

**目标用户画像：**

| 角色 | 典型身份 | 核心诉求 |
|------|---------|---------|
| 采购方/招标方 | 企业采购经理、政采办工作人员 | 流程合规、结果留痕、效率高 |
| 供应商/投标方 | 企业销售、BD经理 | 报名方便、投标安全、结果透明 |
| 评标专家 | 行业专家、内部评审员 | 独立评分、互不干扰、操作简单 |
| 系统管理员 | IT运维、平台运营 | 权限管控、数据安全、配置灵活 |

## 二、核心业务流程

### 2.1 主流程总览

```
项目立项
   │
   ▼
招标公告发布 ──────────────────────────────────────────┐
   │                                                  │
   ▼                                                  │
供应商报名/资质审核                                      │
   │                                                  │
   ▼                                                  │
招标文件购买/下载                                        │
   │                                                  │ 澄清答疑窗口
   ▼                                                  │
投标文件上传（密封）                                      │
   │                                          ◄───────┘
   ▼
开标（定时自动解密）
   │
   ▼
资格审查（招标方）
   │
   ▼
评标（专家独立打分）
   │
   ▼
中标候选人公示（3~7天）
   │
   ├── 无异议 ──► 中标通知书 ──► 合同签署
   │
   └── 有异议 ──► 质疑/投诉处理
```

### 2.2 关键状态机

**招标项目状态：**
```
草稿 → 待审核 → 已发布 → 报名中 → 已截标 → 开标中 → 评标中 → 公示中 → 已定标 → 已归档
                                                                     │
                                                                     └→ 废标/流标
```

**投标书状态：**
```
草稿 → 已提交（加密） → 已开标（解密） → 资格审查中 → 通过/淘汰
```

## 三、功能模块详细设计

### 模块一：项目管理（招标方）

#### 1.1 项目立项

**功能点：**
- 填写项目基本信息：项目名称、采购类型（货物/服务/工程）、预算金额、所属部门
- 上传项目批文/立项文件
- 设置招标方式：公开招标 / 邀请招标 / 竞争性谈判 / 询价 / 单一来源
- 设置时间节点：公告期、报名截止、投标截止、开标时间
- 评标委员会设置：专家人数、专家抽取方式（随机/指定）

**字段设计（立项表单）：**

```
基本信息
├── 项目名称 *
├── 项目编号（自动生成）
├── 采购类型 * [货物/服务/工程]
├── 采购方式 * [公开招标/邀请招标/竞争性谈判/询价/单一来源]
├── 预算金额 *（含币种）
├── 资金来源 [财政资金/自有资金/外资]
└── 项目描述

时间安排
├── 公告开始时间 *
├── 报名截止时间 *
├── 答疑截止时间
├── 投标截止时间 *（= 开标时间）
└── 评标完成时间（预计）

评标设置
├── 评标方法 [综合评分法/最低价法/两阶段评标]
├── 专家人数（奇数，≥3）
├── 专家抽取方式
└── 评分标准模板（可从模板库选择）
```

#### 1.2 招标公告管理

**功能点：**
- 富文本编辑招标公告内容
- 上传招标文件（PDF/Word，支持多文件）
- 设置投标保证金金额及缴纳方式
- 设置资质要求条件
- 预览公告效果（PC端/移动端双预览）
- 提交审核 → 审核通过后自动发布

#### 1.3 供应商资质审核

**功能点：**
- 查看报名供应商列表
- 审查供应商上传的资质材料
- 一键通过/拒绝（填写拒绝原因）
- 批量导出报名名单

#### 1.4 开标管理

**功能点：**
- 定时自动触发开标（到达投标截止时间系统自动解密）
- 开标现场视频记录上传（可选）
- 生成开标记录表（自动）
- 开标结果公示（投标人名称、报价、是否有效）

#### 1.5 定标与通知

**功能点：**
- 查看评标报告
- 确认中标候选人排序
- 发起公示（填写公示期天数）
- 公示期满无异议 → 一键发送中标通知书
- 通知书自动推送（站内信 + 邮件 + 短信）

### 模块二：投标管理（供应商）

#### 2.1 项目浏览与报名

**功能点：**
- 招标大厅：公告列表（支持关键词、采购类型、地区、金额筛选）
- 项目详情页：公告内容、时间安排、招标文件下载、联系方式
- 一键报名：填写报名信息、上传资质材料
- 报名状态跟踪：待审核/已通过/已拒绝
- 投标保证金在线缴纳（对接支付宝/银行转账）

#### 2.2 招标文件购买/下载

**功能点：**
- 免费下载 / 付费购买（可配置）
- 下载记录留痕
- 招标文件变更推送（澄清答疑更新时自动通知已购买方）

#### 2.3 投标文件制作与提交

**功能点：**
- 在线投标文件模板下载
- 分包投标支持（技术标/商务标/报价单独上传）
- 投标文件上传（本地加密，密钥由系统持有，开标前不可解密）
- 投标截止前可撤回重新上传
- 文件完整性校验（MD5）
- 报价输入（支持含税/不含税切换）
- 提交成功后收到电子回执

**投标文件结构：**
```
技术标
├── 技术方案
├── 项目团队
├── 业绩材料
└── 服务承诺

商务标
├── 营业执照
├── 资质证书
├── 财务报表
└── 无犯罪记录证明

报价文件
└── 报价单（标准格式）
```

#### 2.4 开标结果查看

**功能点：**
- 开标后实时查看所有投标人报价
- 查看自己的排名
- 查看中标候选人公示
- 查询中标结果通知

#### 2.5 质疑与投诉

**功能点：**
- 提交质疑（需在公示期内，填写质疑内容+上传证据）
- 质疑进度跟踪
- 对质疑回复不满意 → 升级投诉
- 历史质疑记录

### 模块三：评标管理（评标专家）

#### 3.1 专家库管理（管理员操作）

**功能点：**
- 专家信息录入：姓名、身份证、专业方向、职称、所在单位、联系方式
- 专家资质有效期管理（自动提醒到期）
- 专家黑名单（违规记录）
- 随机抽取专家（按专业、地区条件过滤）
- 专家回避申请处理

#### 3.2 评标任务

**功能点：**
- 评标邀请通知（站内信 + 短信）
- 专家接受/拒绝任务（拒绝需填理由）
- 评标室进入（需验证身份）
- 独立评分界面（互相不可见彼此打分）
- 评分项逐项打分（按评分标准模板）
- 修改理由记录（修改分数需填原因）
- 提交评分确认（不可撤回）

**评标界面设计要点：**

评标界面布局：
┌─────────────────────────────────────────────────────┐
│ [项目名称] 评标室 │ 专家：张三 │ 时间：倒计时      │

│  ─────────     │  ─────────────────────────────     │
│  ● 公司A (5/8) │  技术方案 (30分)                   │
│  ○ 公司B (0/8) │  ┌──────────────────────────────┐ │
│  ○ 公司C (0/8) │  │ 评分项1: 方案完整性 (10分)    │ │
│                │  │ [得分输入框] 评语___________  │ │
│                │  └──────────────────────────────┘ │
│                │  ┌──────────────────────────────┐ │
│                │  │ 评分项2: 技术先进性 (10分)    │ │

#### 3.3 评标报告

**功能点：**
- 系统自动汇总各专家得分（去掉最高最低分后取平均，可配置）
- 生成评标报告（自动填写，支持专家确认签字）
- 自动排名
- 导出评标报告 PDF

## 四、非功能性需求

### 4.1 安全合规

| 要求 | 实现方式 |
|------|---------|
| 投标文件保密 | 上传后 AES-256 加密，开标时间前密钥不下发 |
| 操作全留痕 | 所有关键操作记录操作日志（操作人、时间、IP、内容变更） |
| 防串通 | 评标专家互相隔离，系统不展示其他专家分数 |
| 防篡改 | 投标文件 MD5 校验，存储不可变 |
| 数据权限 | 复用现有 @DataPermission 注解体系 |

### 4.2 性能要求

| 场景 | 要求 |
|------|------|
| 公告列表页 | 首屏 < 2s，支持 10000 条数据分页 |
| 投标文件上传 | 支持单文件 500MB，断点续传 |
| 开标解密 | 100 家投标人同时解密 < 30s |
| 并发评标 | 50 名专家同时评分无感知延迟 |

### 4.3 可用性

- 系统可用性 SLA：99.9%（每月不超过 44 分钟宕机）
- 关键流程（开标、投标截止）前后 30 分钟禁止维护窗口
- 移动端响应式支持（H5）

### 5.3 交互设计原则

1. **关键操作二次确认：** 投标提交、废标、定标等不可逆操作，必须二次弹窗确认
2. **倒计时显眼展示：** 截标时间在所有相关页面固定显示，临近截标时变红 + 震动动效
3. **状态可感知：** 每个项目状态用颜色标签区分，操作引导清晰
4. **防误操作：** 评分提交后置灰所有修改按钮，修改需走"申请修改"流程
5. **移动端适配：** 供应商端优先适配移动端（报名、查看结果为高频移动场景）

## 六、集成到 AI-DDD 项目的模块规划

### 6.1 聚合根设计

```
BidProject（招标项目）聚合根
├── 核心标识：projectId, projectNo（自动生成）
├── 状态机：ProjectStatus（草稿→发布→报名→截标→开标→评标→公示→定标→归档）
├── 值对象：TimelineConfig, EvaluationConfig, BudgetInfo
└── 领域事件：
    ├── ProjectPublishedEvent（项目发布）
    ├── BiddingClosedEvent（截标）
    ├── ProjectOpenedEvent（开标）
    └── WinnerAnnouncedEvent（定标）

Bid（投标书）聚合根
├── 核心标识：bidId, projectId, supplierId
├── 状态机：BidStatus（草稿→已提交→已开标→已审查→通过/淘汰）
├── 值对象：BidFiles（加密文件集合）, BidPrice
└── 领域事件：
    ├── BidSubmittedEvent（投标提交）
    └── BidDecryptedEvent（开标解密）

EvaluationTask（评标任务）聚合根
├── 核心标识：taskId, projectId, expertId
├── 状态机：TaskStatus（待接受→已接受→评分中→已提交）
└── ScoreSheet（评分表，值对象集合）

Supplier（供应商）聚合根
├── 供应商资质、资质有效期
├── 黑名单状态
└── 历史中标记录
```

### 6.2 工作流集成

复用现有 Activiti 7.1.0 引擎，映射以下审批流程：

| 业务流程 | 工作流节点 |
|---------|-----------|
| 立项审批 | 发起人 → 部门负责人 → 采购委员会 |
| 公告审核 | 采购员 → 审核员 → 发布 |
| 定标审批 | 评标结果 → 采购负责人 → 分管领导 → 发通知书 |
| 质疑处理 | 收到质疑 → 指定处理人 → 回复 → 申请方确认 |

## 八、关键风险与应对策略

| 风险 | 等级 | 缓解方案 |
|------|------|---------|
| 开标前文件泄露 | 高 | 服务端 AES-256 加密 + 密钥分片，开标时间自动触发解密 |
| 评标专家串通 | 高 | 独立评分界面，评标期间屏蔽通讯，IP 审计日志 |
| 操作留痕不足 | 中 | 所有写操作接入现有操作日志模块（OperationLog） |
| 高并发开标场景 | 中 | 消息队列异步解密，Redis 分布式锁防重复触发 |
| 多租户数据隔离 | 高 | 所有表加 `tenant_id`，复用现有 @DataPermission |
| 投标截止时系统故障 | 高 | 关键时间点前后 30 分钟禁止维护，自动告警 |

## 九、名词解释

| 术语 | 说明 |
|------|------|
| 开标 | 在规定时间公开投标文件，解密并展示各方报价 |
| 定标 | 最终确定中标单位 |
| 公示 | 中标结果对外公开展示，接受社会监督 |
| 质疑 | 供应商对招标过程提出异议（须在公示期内） |
| 废标 | 因投标人数量不足或其他原因导致本次招标无效 |
| 流标 | 无人投标或所有投标文件均不符合要求 |
| 综合评分法 | 对投标文件的技术、商务、报价等方面综合打分 |
| 最低价法 | 满足技术要求的前提下，报价最低者中标 |
| 评标基准价 | 所有有效报价的平均值，用于计算商务评分 |
